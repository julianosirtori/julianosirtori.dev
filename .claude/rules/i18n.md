## Both languages, always

- Every copy change in `src/locales/en/*` has a matching change in `src/locales/pt/*` (same key path)
- Hardcoded UI strings are a smell. Add a key under the appropriate scope (`global`, `home`, `blog`, `playground`, ...) and read via `useTranslations` / `getTranslations`
- Blog content under `content/en/` and `content/pt/` should share filename slugs so URLs match across languages

## Reading translations

```ts
// Client component
import { useTranslations } from "next-intl";
const t = useTranslations("home");
return <p>{t("subtitle")}</p>;

// Server component
import { getTranslations } from "next-intl/server";
const t = await getTranslations("blog");

// generateMetadata
const t = await getTranslations({ locale: lang, namespace: "playground" });
```

Always pass the locale explicitly in `generateMetadata` and other server-only paths where the request locale isn't set yet.

## Voice guide (binding)

See `docs/voice-guide.md`. Highlights:

- **First person, active voice.** "Construo interfaces", not "Interfaces são construídas".
- **No em dashes.** Use periods, commas, or restructure.
- **No exclamation marks** in slogans or hero copy.
- **No clichés**: avoid "perfeito", "incrível", "soluções", "experiências envolventes", "create interactions", "let's dive into".
- **Specific verbs**: construir, escrever, errar, refatorar, depurar.
- **Empathy hooks**: "muito se fala em X", "quem nunca, né?", "se você se encontra neste cenário".
- **Rewrite, don't translate** between EN and PT. Each language has its own phrasing for the same intent.

## URL structure

```
/{lang}/{page}
/en/blog              → English blog
/pt/blog              → Portuguese blog
/en/blog/post-slug    → English post
/pt/blog/post-slug    → Portuguese post
```

`src/middleware.ts` (next-intl) handles routing. Use `Link` and `useRouter` from `@/locales/navigation` so locale prefix is preserved.

## Locale file structure

```
src/locales/{en,pt}/
├── global.json      header, footer, social, kbar (cmdk keys live here)
├── home.json
├── blog.json
├── about.json
├── projects.json
├── contacts.json
├── guestbook.json
├── now.json
└── playground.json
```

Each language's `index.ts` re-exports every JSON in the same order so `importLocale` resolves consistently.
