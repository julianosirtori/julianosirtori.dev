Este é o primeiro artigo de uma série intitulada QuickBytes. Serão artigos curtos abordando uma variedade de temas. O primeiro deles é sobre como criar um monorepo com pnpm workspaces.

Para criar um monorepo com pnpm, precisaremos criar um arquivo chamado `pnpm-workspace.yaml` dentro do nosso repositório.

```shell
touch pnpm-workspace.yaml
```

Dentro do `pnpm-workspace.yaml`, vamos definir nossos pacotes. Ele terá uma estrutura semelhante a esta:

```yml
packages:
  - 'app'
  - 'api'
  - 'utils'
  - 'services'
```

Agora podemos fazer referência aos nossos pacotes do workspace da seguinte forma:

```json
{  
	"dependencies": {  
		"@repo/utils": "workspace:*",
		"@repo/services": "workspace:*"  
	}  
}
```

Você pode encontrar mais detalhes na [documentação do pnpm](https://pnpm.io/workspaces). E como podemos observar, muitos dos maiores frameworks do mercado adotam essa abordagem.

### Bônus 

Para otimizar nosso fluxo de desenvolvimento, podemos integrar o [Turborepo](https://turbo.build/repo/docs/getting-started/add-to-project), uma ferramenta que simplifica a execução de scripts em vários `packages.json` simultaneamente.
